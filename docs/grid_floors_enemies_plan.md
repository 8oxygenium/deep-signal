# PUDDING INFERNO GRID — 階層化＆敵ロスター プラン

対象: `games/pudding-inferno-grid-prototype/`（現 GRID v0.2.3）
方針: **検証ファースト・小さく**。1階につき新要素1つ。深さ＝難しさでなく「深さ＝新しい敵に会う」。最終審査は娘さん（小3）実機。理不尽（予告なし即死・過度なフォグ）を避ける。

## 決定事項（2026-06-27・隆郎さん裁定）
- **1ラン = 全5階**：B1〜B4 が通常階、**B5 = ボス階**。
- **敵 = 3種**：騎士（既存）＋牙（はや）＋肉塊（おも）。遠距離砲は今回入れない。
- **全敵に「向き（プレイヤーをにらむ）」**を付ける（プリンの faceX/faceY と同手法）。
- **死亡時 = 2モードを難易度設定で用意**（設定UIは後イテ）：
  - ハード（本格ローグライク）= 死んだら **B1 から**（パーマデス）。
  - やさしい = **死んだ階からリトライ**。
  - MVP実装では内部フラグ `difficulty` で両対応にしておき、UIは後付け。

- 出口の意味を変更：今の「出口踏む＝即CLEAR」→ **「次の階へ降りる」**。最終 B5 クリアのみ CLEAR。
- 降りる時はプレイヤーの状態（HP・ハット・クリーム）を引き継ぐ。位置・敵・罠・フォグは新規生成。

## 敵ロスター（"ステータス違い"でなく"行動が違う"）
既存 `doEnemyTurns()`（BFS追跡／隣接で warn◇→次ターン攻撃）に種別分岐を足す。`enemy.type` を追加。

| 敵 | type | HP | 行動 | グラフィック |
|---|---|---|---|---|
| 鎧騎士（既存） | `knight` | 2 | 追跡・隣接で◇予告→次ターン攻撃 | 暗赤の丸＋鎧十字＋炎。**いかり目のバイザー追加（にらむ）** |
| 牙（はや） | `fang` | 1 | 追跡・**予告なしで隣接した瞬間に噛む**（脆い） | 小さく・骨色・ギザギザの牙。素早く危険そう |
| 肉塊（おも） | `blob` | 3 | **2ターンに1回だけ動く**・通路を塞ぐ壁役 | 大きい・ボコボコの肉塊・重い |

対処法が三者三様（予告して殴れる／予告なしだが脆い／鈍いが硬い）＝小6指摘「キャラ替えだけは二番煎じ」回避。

## 敵の向き
全敵、毎フレーム「敵→プレイヤー」方向を sign で出し、目/バイザーをその方向へオフセット。
- 体内ダンジョンで「魔物がこっち見てる」圧。
- 牙が今どっちを狙ってるか一目で分かる＝予告なし噛みのフェア化。
